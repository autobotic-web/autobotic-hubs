1 Why a matrix?

A Raspberry Pi has only so many GPIO pins. Wiring one pin per button runs out fast. A matrix shares wires so a handful of pins read lots of buttons.

Arrange buttons in a grid. Every button sits where a row wire crosses a column wire. To name any button you only need its row and its column — just like a seat at the cinema (Row C, Seat 7).

🧰 Vocabulary Matrix — buttons in a grid of rows and columns (like a spreadsheet). Row — here, an input pin that reads a press (GPIO 19, 13). The ear. Column — an output pin we pulse to scan (GPIO 22, 24, 25, 23). The spotlight. Pull-up — keeps a pin HIGH (1) until something pulls it LOW (0). A spring that holds a door shut until you push it. Active-LOW — "on" means driven to 0 / LOW. Sounds backwards, but it's how this matrix works.

2 How scanning works

The Pi can't read all 8 buttons at once. Instead it scans: light up one column at a time and check which row reacts.

The four steps the Pi repeats, fast:

1️⃣ Set every column HIGH (off).   2️⃣ Drive one column LOW (active — that's column.off()).   3️⃣ Check each row: a pressed button pulls its row LOW.   4️⃣ Active column + the low row = the exact button. Then set that column back HIGH and move on.

The big gotcha: always set the column back HIGH (column.on()) before scanning the next one. Leave it LOW and the next column gets misread — you'll see ghost presses.

🐛 Bug Hunt — this scan reports presses that never happened. Click the broken line:

row = Button(19, pull_up=True)
column = OutputDevice(22, initial_value=True)
column.on() # activate the column
if row.is_pressed:
    print("SW1 pressed!")
Why that line? column.on() drives the column HIGH, which turns it off — so it never activates and the reading is meaningless (or stuck). To activate a column for scanning you need column.off() (LOW). It's the active-LOW trap: the method name feels opposite to what it does.
