1 Frames, pixels & OpenCV

A camera gives you frames — one still image at a time, many per second. To a computer each frame is a grid of pixels (numbers). OpenCV (cv2) is the toolbox for working with them.

The core loop: open the camera (VideoCapture), read a frame (cap.read() → ret, frame), show it (imshow), repeat. Then you process the frame — convert colours, find shapes, or run a trained detector.

🐛 Bug Hunt — the program crashes with "NoneType" right after reading. Click the line that's used too early:

ret, frame = cap.read()
cv2.imshow("Feed", frame)  # no ret check!
cv2.waitKey(1)
Why that line? If the read fails, frame is None, so imshow crashes. Add a check first: if not ret: break. A failed frame is normal sometimes — guarding for it keeps the program alive.
