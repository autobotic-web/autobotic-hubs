1 Characters, lines & I²C

The LCD 1602 shows 16 characters across, 2 lines down. Each spot holds one character. An I²C backpack chip lets the Pi drive it with just two signal wires (SDA + SCL).

Every I²C device has an address (the LCD's is often 0x27 or 0x3F). You find it with i2cdetect -y 1 in the terminal, and your code must use that exact address — like a house number on a street of chips.

🧰 Vocabulary LCD 1602 — a 16×2 character screen for short text. I²C — a two-wire bus (SDA + SCL) that talks to many chips. Address — each I²C device's ID number (e.g. 0x27); the code must match it. cursor_pos — (row, column): where the next text is written. (0,0)=line 1. write_string — the RPLCD method that puts text on the screen.

🐛 Bug Hunt — the code runs with no error but the screen stays blank. i2cdetect shows the LCD at 0x3F. Click the broken line:

lcd = CharLCD("PCF8574", 0x27, port=1, cols=16, rows=2)
lcd.write_string("Hello!")
time.sleep(3)
Why that line? The code uses 0x27 but the display answers at 0x3F, so the text is sent to the wrong address and nothing shows. Fix: use 0x3F (the address i2cdetect reported). Always check the real address first.
