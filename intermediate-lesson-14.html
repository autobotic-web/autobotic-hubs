1 Colour is three numbers

Each LED on the strip is really three tiny lights — Red, Green, Blue. Mix how much of each (0–255) and you get any colour. (255,0,0) is red; (255,255,0) is yellow.

The strip is addressable: the Pi sends colour data to LED #0, which passes it to #1, and so on down one wire — so one GPIO pin drives the whole strip. You set the colours, then call pixels.show() to push them out.

🔬 Mix a colour

Tap a preset to fill the strip — see the (R,G,B) behind each colour, just like pixels.fill():

pixels.fill((0, 0, 0)) → off
🧰 Vocabulary WS2812B / NeoPixel — an addressable RGB LED strip (each LED its own colour). Addressable — you can set each LED individually; data passes down the chain. (R, G, B) — a colour as three numbers 0–255 (Red, Green, Blue). pixels.show() — pushes your colour changes out to the strip. sudo — run with admin power, needed for the strip's precise timing.

🐛 Bug Hunt — the colour is set but the strip never lights. Click the missing-step line that needs fixing:

pixels = neopixel.NeoPixel(board.D18, 8, auto_write=False)
pixels.fill((0, 0, 255))
time.sleep(2)  # strip never updates...
Why that line? After fill() there's no pixels.show(), so the blue never reaches the LEDs — the program just sleeps with the strip unchanged. Add pixels.show() before the sleep. With auto_write=False, no show() means no light.
