1 What's an LDR?

LDR stands for Light Dependent Resistor. Its resistance drops in bright light and rises in the dark — so the amount of light changes how easily electricity flows through it.

The Raspberry Pi has no analog input — it can't read a smooth "brightness number" on its own (an Arduino can; the Pi can't). So the OiviO board compares the LDR to a threshold and hands the Pi one clean bit:

0 = light  •  1 = dark

Want a real number (a bit dim vs pitch black)? You'd add an ADC chip like the MCP3008 to convert the analog signal — but for light-vs-dark, one bit is all we need.

🧰 Vocabulary LDR — a resistor whose resistance changes with light (low in bright, high in dark). Digital signal — a value that is only ever 0 or 1 (LOW or HIGH). A light switch, not a dimmer. Threshold — the cut-off that decides "light" from "dark". The on-board knob sets it. ADC — Analog-to-Digital Converter; a chip that turns a smooth signal into numbers. Analog — a value that varies smoothly (like a volume dial), the opposite of digital.

2 Light as a decision

A sensor on its own is useless — the magic is reacting to it. The pattern: read the value, compare it to a threshold, then act. That's perceive → reason → act.

Your robot's rule will be simple: if it's dark (value 1), do something; if it's light (value 0), don't. One comparison is the simplest "decision" a machine can make.

🐛 Bug Hunt — this night-light comes on in daylight instead of the dark. Click the broken line:

ldr = DigitalInputDevice(4)
led = LED(21)
if ldr.value == 0:   # meant to detect dark
    led.on()
Why that line? On this board value == 0 means light, so the LED lights up in daylight — backwards. A night-light should test value == 1 (dark). It's the classic polarity mix-up: always check in both bright and dark and flip the comparison if it reacts the wrong way.
