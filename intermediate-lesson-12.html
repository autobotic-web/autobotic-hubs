1 A grid of pixels

An 8×8 matrix is 64 LEDs — 8 rows by 8 columns. Each LED has an address: an (x, y) coordinate. (0, 0) is the top-left; (7, 7) is the bottom-right.

Wiring 64 LEDs by hand is impossible, so the MAX7219 driver chip handles them all over SPI (three signal lines: DIN/MOSI, CLK, CS). You just say "light the LED at (x, y)" and the chip does the rest.

⚠ Two setup steps: turn SPI on (sudo raspi-config → Interface Options → SPI → Enable → reboot) and install luma (pip3 install luma.led_matrix). Note: while SPI is on, the green LED (GPIO 7) won't work — it shares that pin. That's expected.

🧰 Vocabulary LED matrix — a grid of LEDs (here 8×8 = 64) you switch on/off individually. Pixel — one dot of a picture; here, one LED. (x, y) coordinate — an LED's address: x across, y down. (0,0) is top-left. SPI — a fast way chips talk using a few lines (DIN/MOSI, CLK, CS). MAX7219 — the driver chip that controls all 64 LEDs for you.

with canvas(device) as draw:
    draw.point((0, 0), fill="black")
time.sleep(2)
Why that line? fill="black" means "off", so the LED is drawn dark — nothing appears. To light it, use fill="white". On a single-colour matrix, white = on and black = off; it's an easy slip to make.
