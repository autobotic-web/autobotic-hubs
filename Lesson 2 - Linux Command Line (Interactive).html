1 The filesystem is a tree

Every file lives in a folder, and every folder lives in another folder, all the way up to the root /. Your stuff lives in your home folder, /home/pi.

/            ← root: the very top
└── home
    └── pi   ← your home folder (~)
        ├── Documents
        ├── Downloads
        └── python_practice
~ is a shortcut for your home folder. cd .. goes up one level, cd / jumps to the root.

The bits after the command (-l, -a) are called flags or options — almost every command takes them. ls -l shows extra detail (size, date), and ls -a reveals hidden files whose names start with a dot, like .bashrc. Real-life example: settings files are usually hidden, so ls -a is how you find them.

Moves you into the Documents folder. cd .. goes up one level; cd / goes to the root.

Lost a file? Use find

find ~ -name <filename> searches your entire home folder (~) and every folder inside it, then prints exactly where the file lives. Real-life example: you saved the OiviO bridge program but can't remember which folder it's in — find ~ -name autobotic_bridge.py tells you instantly, instead of clicking through folders one by one. Use * as a wildcard to match more: find ~ -name "*.py" lists every Python file in your home folder.

🆘 Get unstuck on your own: forgotten what a command does or which flags it takes? Ask the command itself — man ls opens its manual (press q to quit), and most commands also accept --help (e.g. ls --help). Real engineers look things up constantly — knowing how to find the answer beats memorising everything.

4 Creating & managing files

cat prints a file's contents straight to the screen — a fast way to peek inside a script or text file without opening an editor. Real-life example: cat distance.py to double-check your code is what you think it is before you run it.

rm deletes for good — there's no recycle bin on the command line. Double-check before you press Enter. And never put spaces in file names; use my_file.txt instead.

6 Installing & removing software

sudo gives you administrator power. apt-get installs and removes programs (needs internet):

Run your programs & change Pi settings

This is how you run code from the terminal — exactly how you'll start every program in the next lessons, and how you start the OiviO bridge: python3 autobotic_bridge.py. Press Ctrl + C to stop a program that's running.

sudo raspi-config opens the Pi's settings menu. You'll use it to switch hardware on or off — for example turning SPI off so the green LED works (Interface Options → SPI). Most setting changes need a restart afterwards:

sudo = "super user do". It runs a command as the administrator, so use it carefully — it can change important system files.
